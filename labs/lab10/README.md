# Lab 10: NER Pipeline + Hybrid Rules (UA Datasets)

## 1. Який корпус та Evaluation Set використано
Використано репрезентативну вибірку україномовних новин та зведень (UA Datasets). Для тестування підготовлено **Evaluation Set із 15 речень**, які містять складні крайові випадки: військові абревіатури, законодавчі акти та складні формати запису грошей.

## 2. Який pipeline запущено
Базовим пайплайном виступає **spaCy** з офіційною українською моделлю `uk_core_news_sm`. Для гібридного розширення використано компонент `EntityRuler`.

## 3. Які rules додано
Впроваджено 3 типи правил для вирішення специфічних проблем українського тексту:
1. **Словники / PhraseMatcher (`MILITARY`, `ORG`):** Перелік державних та військових абревіатур (ЗСУ, ГУР, ТЦК, НАБУ), щоб уникнути їх пропуску або хибної класифікації.
2. **Token Regex (`LAW`):** Гнучкий шаблон для вилову нормативно-правових актів ("законопроєкт №10449", "указ", "постанова").
3. **Boundary format (`MONEY`):** Правило для об'єднання числівників, порядків та валют ("100 млн грн") у єдину неподільну сутність.

## 4. Що baseline знаходив добре
Статистична модель чудово впоралася з розпізнаванням класичних сутностей: імен політиків та діячів (`PER`), а також назв українських міст і регіонів (`LOC`).

## 5. Що baseline пропускав
* **Абревіатури:** Повністю ігнорував або плутав військові та антикорупційні органи.
* **Закони:** Пропускав усі законодавчі акти, оскільки не має відповідного лейблу.
* **Гроші:** Масово робив Boundary Errors, витягуючи лише цифру (як `CARDINAL`) замість повної суми валюти.

## 6. Що rules покращили
Гібридний шар повністю "вилікував" ці проблеми:
* Розірвані гроші тепер ідеально склеюються у сутність `MONEY`.
* З'явилося 100% покриття законів та військових органів із нашого словника.
* Кількість Exact Match правильних сутностей на тестовому наборі зросла втричі (з 7 до 22).

## 7. Які проблеми ще не вирішені
Для повноцінної роботи потрібно враховувати складну українську флективність (відмінки), додавши лематизацію перед застосуванням словникових правил. Також залишається проблема розпізнавання зовсім нових багатослівних організацій, що вимагає донавчання моделі (Fine-Tuning NER).
