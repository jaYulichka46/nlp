# Лабораторна робота №12 — Tool-grounded Single-Agent

### 1. Опис Use Case
Метою роботи є створення **News Analyst Assistant** — агента, який аналізує короткі тексти українських новин і автоматично витягує з них ключову інформацію: категорію новини та основні іменовані сутності (локації, організації, особи).

### 2. Архітектура та Інструменти (Tools)
Система побудована за архітектурою ReAct (Reason + Act). Замість того, щоб покладатися виключно на знання моделі, агент використовує детерміновані Python-функції (інструменти) для точного пошуку даних та їх класифікації. Усі виклики суворо фіксуються через `ToolLogger` у файл `tool_logs_lab12.jsonl`.

Реалізовано 2 інструменти:
1. `extract_key_entities`: Використовує словник (rule-based підхід) для пошуку відомих українських та світових локацій, організацій (ЗСУ, НАТО) та політиків.
2. `categorize_news`: Використовує ключові слова для класифікації новини за категоріями (War/Military, Politics, Economy, Sports, General).

### 3. Базові Метрики (на 10 Test Cases)
* **Tool call success rate:** 100.0% (усі функції Python відпрацювали без системних збоїв).
* **Average tool calls per task:** ~2.0 (Агент системно викликав обидва інструменти для повноти картини).
* **Відсоток задач із корисними викликами:** 80% (успішно класифіковано та витягнуто дані з цільових новин).
* **Зайві виклики (Unnecessary calls):** 20% (Агент викликав інструмент пошуку політичних/військових сутностей для прогнозу погоди та клікбейту).

### 4. Аналіз Baseline vs. Agent
Базова модель (Baseline LLM), аналізуючи новини, часто видавала неструктуровані відповіді у вигляді есе, а також могла самостійно вигадувати "нові" категорії, не передбачені системою. 
Tool-grounded Agent повністю усунув цю проблему. Завдяки інструменту `categorize_news`, категорія завжди була строго з нашого списку. Агент також чітко оформлював знайдені сутності. Проте виявлено обмеження: жорсткий словник в `extract_key_entities` не дозволив знайти спортивні сутності (футболістів), оскільки їх не було в базі.

### 5. Головний висновок та напрямки покращення
Агентний підхід робить аналіз новин значно більш передбачуваним і структурованим, що ідеально для наповнення баз даних. Головний напрямок для покращення — замінити хардкод-словник в інструменті екстракції на виклик справжньої NER-моделі (наприклад, SpaCy), щоб знаходити сутності контекстно, а не лише за списком. Також варто навчити агента зупинятися (Early Exit), якщо новина класифікується як "General/Unknown".
