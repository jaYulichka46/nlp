# Лабораторна робота 4: Rule-based Information Extraction (News Dataset)

## 1. Які 3 типи полів ви витягуєте
* **MONEY_AMOUNT**: Грошові суми, бюджети та фінансові втрати з конвертацією текстових множників (млн, млрд) у числа.
* **DATE_EVENT**: Дати подій у цифровому (DD.MM.YYYY) та текстовому (напр., "15 березня 2023") форматах.
* **UA_AGENCY**: Українські державні установи та відомства (нормалізація абревіатур до повних офіційних назв).

## 2. Які правила/словники використали
* Створено словник `resources/ua_agencies.json` для мапінгу абревіатур (ЗСУ, НАБУ, Кабмін) у повні назви. Пошук реалізовано з використанням негативних lookbehind/lookahead для кирилиці `(?<![а-яА-Яіїєґ])`, щоб знаходити точні збіги слів.
* Написано Regex для дат, який підтримує текстові місяці через словник `MONTHS_MAP` та автоматично підставляє поточний рік, якщо його не вказано.
* Написано Regex для грошей, який парсить суми типу "1.5 млрд грн" або "$50 тисяч", ігноруючи відсотки чи кількість людей (наприклад, "10 тисяч мітингувальників").

## 4. Топ-5 edge cases у нашому корпусі
1. **Текстові множники з дробами**: "1.5 млрд грн" -> перетворено на `1500000000.0`.
2. **Абревіатури всередині слів**: Запобігання витягуванню "СБУ" зі слова "МА**СБУ**Д" (вирішено через межі слів).
3. **Дати з текстовими місяцями**: "15 березня 2022 року" -> нормалізовано у `15.03.2022`.
4. **Негрошові числа**: "10 тисяч людей" -> проігноровано, оскільки немає індикатора валюти.
5. **Дати без року**: "Дедлайн 10.05" -> підставлено поточний рік (`10.05.2026`).
