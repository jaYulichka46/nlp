# Лабораторна робота 3: Lemma/POS baseline (UA-datasets / Новини)

**Напрям:** A (Класифікація текстів / Topic Modeling)  
**Інструмент:** Stanza (uk)

## Що зроблено
1. Написано скрипт `src/ling_features.py` для генерації лем та POS-тегів з українських новин.
2. Створено `tests/ling_edge_cases.jsonl` з 10 новинними Edge Cases (абревіатури, складні прізвища, омонімія) та проведено Error Analysis.
3. Побудовано два Baseline-підходи (TF-IDF + Logistic Regression) для порівняння якості класифікації на сирому та лематизованому тексті.

## Основні цифри "до/після" (на вибірці 1000 записів)
* **Baseline 1 (Raw Text):** Accuracy = 0.4500, Macro-F1 = 0.2842, Розмір словника = 5000 ознак.
* **Baseline 2 (Lemma Text):** Accuracy = 0.4500, Macro-F1 = 0.2842, Розмір словника = 4830 ознак.

## Висновок
**Рішення:** Використовувати лематизацію для подальшого пайплайну класифікації.
**Обґрунтування:** Хоча лематизація локально помиляється на специфічних абревіатурах (наприклад, ЗСУ), для задачі класифікації вона нормалізує словоформи (зводить "закон", "закону" до однієї ознаки). Як видно з метрик, лематизація **не погіршила точність моделі (Accuracy залишилась 0.4500)**, але при цьому успішно **зменшила розмір матриці TF-IDF на 170 ознак (з 5000 до 4830)**. Це зменшує зашумленість даних і робить модель більш оптимізованою.
