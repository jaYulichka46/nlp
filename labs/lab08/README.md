# Lab 8: Topic Modeling (UA News)

1. **Який корпус аналізували:** Українські новини (`ua_news_processed.csv`), відфільтровані від коротких текстів (залишено $\ge 10$ слів, загалом ~120k документів).
2. **Які моделі запускали:** LSA (`TfidfVectorizer` + `TruncatedSVD`) та LDA (`CountVectorizer` + `LatentDirichletAllocation`).
3. **Які k перевіряли:** $k=5$ та $k=8$.
4. **Які теми вийшли найкращими:** Спорт (футбол, турніри), Фінанси (курс валют на міжбанку), Війна/АТО (зведення з фронту). Ці теми найкраще сформувала LSA.
5. **Які теми вийшли поганими:** Тема зі стоп-слів та загальних займенників (`якщо`, `може`, `так`) і тема доменного шуму (`еспресо`, `tv`, `повідомляє`). Обидві згенеровані LDA.
6. **Що показало порівняння LSA vs LDA:** LSA дала набагато чистіші та більш читабельні результати. TF-IDF допоміг "приглушити" всюдисущі новинні штампи. LDA виявилася надто чутливою до шаблонів та копірайтів ЗМІ.
7. **Чи topic modeling взагалі корисний для вашого корпусу:** Так, для автоматичного розбиття новин на рубрики (наприклад, для рекомендаційної системи), але лише за умови глибокої попередньої очистки специфічних журналістських кліше.
