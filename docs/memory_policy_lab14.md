# Memory & Knowledge Policy (Lab 14 - Ukrainian News)

**1. Що зберігається в state**
У `FlowState` зберігаються лише дані, що стосуються поточного кейсу (task_id): сирий текст новини, очищений текст, присвоєний маршрут, сирі виводи від LLM (`extracted_data`), результати валідації та масиви попереджень і помилок.

**2. Що не зберігається**
Не зберігається історія попередніх новин (Flow є stateless між різними запусками). Не зберігаються ваги моделей або конфігурації підключення до LLM всередині об'єкта стану.

**3. Які intermediate outputs можуть передаватися між етапами**
Етап `Execute` передає згенерований словник у `state.extracted_data`. Етап `Validate` не змінює ці дані, але створює об'єкт `state.validation_result`, який читає етап `Fallback` для прийняття рішення про відновлення.

**4. Як логуються помилки**
Помилки (наприклад, `JSONDecodeError`) не викликають падіння скрипта. Вони акумулюються у масиві `state.errors` (для критичних збоїв) та `state.warnings`. Ці масиви долучаються до фінального експорту для подальшого аналізу.

**5. Як працює knowledge / schema registry**
Всі схеми (наприклад, `NEWS_ANALYSIS_SCHEMA`) та списки ключових слів існують як константи в окремому файлі `src/schemas.py`. Етапи Flow лише імпортують їх для читання (Read-only).

**6. Які файли є read-only knowledge**
* `NEWS_ANALYSIS_SCHEMA` (сувора JSON-схема вимог).
* `WEATHER_KEYWORDS` (маркери прогнозу погоди).
* `SPAM_KEYWORDS` (маркери клікбейту та спаму).

**7. Як запобігається state pollution**
Для кожної нової новини створюється абсолютно новий екземпляр класу `FlowState(raw_text)`. Змінні не "перетікають" між різними обробками. Якщо етап `validate` бракує дані, вони позначаються невалідними, і Fallback може їх перезаписати.

**8. Які дані не можна логувати**
У логах (`flow_logs_lab14.jsonl`) заборонено зберігати API-ключі або системні шляхи локальної машини.
