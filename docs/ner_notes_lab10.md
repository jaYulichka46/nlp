# NER Notes: Lab 10 (UA Datasets)

## 1. Який pipeline вибрано
Для аналізу українського корпусу обрано бібліотеку **spaCy** з офіційною моделлю **`uk_core_news_sm`**. Цей вибір обумовлений високою швидкістю роботи, підтримкою української морфології та гнучкістю компонента `EntityRuler` для побудови гібридних систем.

## 2. Які сутності він уміє з коробки
Базова українська модель розпізнає лише 4 типи сутностей:
- `PER`: Особи (Володимир Зеленський, Сирський).
- `LOC`: Географічні об'єкти (Київ, Авдіївка).
- `ORG`: Організації (Кабмін, Верховна Рада).
- `MISC`: Різні інші сутності.

## 3. Які доменні сутності для вашого корпусу важливі
У контексті українських новин та військово-політичних зведень критично важливими є:
- **`MILITARY`**: Збройні сили, спецслужби та підрозділи (ЗСУ, СБУ, ГУР, ТЦК).
- **`LAW`**: Нормативно-правові акти (законопроєкти, постанови, укази).
- **`MONEY`**: Повні грошові суми з урахуванням масштабу (млн/млрд грн).

## 4. Які правила ви додали
Впроваджено гібридний шар через `EntityRuler`:
1. **Словник абревіатур:** Пряме відображення частотних скорочень (ЗСУ, ДБР, НАБУ) на відповідні мітки (`MILITARY` / `ORG`).
2. **Regex для документів:** Шаблон `[назва документа] + [символ №] + [номер]`.
3. **Boundary Fix для валют:** Патерн, що об'єднує числівник, текстовий множник та назву валюти в одну сутність, запобігаючи розриву межі сутності (Boundary Error).

## 5. Що стало краще після правил
- **Recall для абревіатур:** Показник зріс з майже нульового рівня до 100% для ключових органів.
- **Точність фінансових даних:** Грошові суми тепер витягуються як цілісні об'єкти ("500 тисяч доларів"), а не як окремі цифри.
- **Новий клас LAW:** Система почала ідентифікувати законопроєкти та постанови, які раніше повністю ігнорувалися.

## 6. Які помилки залишилися
- **Складні назви підрозділів:** Багатослівні назви (напр. "Окрема президентська бригада імені гетьмана Богдана Хмельницького") можуть розпізнаватися лише частково через складну структуру.
- **Амбівалентність абревіатур:** Рідкісні скорочення можуть плутатися з іменами власними, якщо вони складаються з 3-4 літер.

## 7. Що б ви фіксили далі
- **Морфологічна варіативність:** Додав би лематизацію перед спрацюванням правил словника, щоб краще ловити назви у непрямих відмінках.
- **Розширення класів:** Виділення сутності `WEAPON` (типи озброєння: HIMARS, дрони, ППО).
- **Training:** Збір та розмітка специфічного датасету для донавчання (Fine-tuning) нейронного шару NER на текстах військових зведень.


## Lab 10 Updates: NER & Hybrid Rules (UA Datasets)

* **Які типи сутностей реально важливі:** Для українського суспільно-політичного корпусу найважливішими виявилися доменні сутності `MILITARY` (Збройні сили, спецслужби, підрозділи), `LAW` (закони, укази, постанови), а також `MONEY` у специфічних українських форматах запису.
* **Чи стандартний NER pipeline достатній:** Абсолютно недостатній. Базова українська модель (`uk_core_news_sm`) добре знаходить імена (`PER`) та міста (`LOC`), але "сліпа" до абревіатур і не має категорій для юридичних документів.
* **Які доменні сутності виявилися проблемними:** Найбільше проблем виникло з абревіатурами (ЗСУ, ТЦК, ГУР) — модель їх пропускає або хибно класифікує. Критичною виявилася категорія `MONEY`: базова модель систематично розриває суми (boundary errors), витягуючи лише цифри і відкидаючи "млн грн" чи "тисяч доларів".
* **Чи hybrid rules дали відчутний виграш:** Так, виграш колосальний. Використання `EntityRuler` зі словниками та RegEx-патернами дозволило "склеїти" розірвані грошові суми, стовідсотково знаходити закони та військові абревіатури.
* **Які типи помилок залишаються:** Залишається складність із сильно змінюваною українською морфологією (якщо слова стоять у непрямих відмінках, які не покриті словником) та багатослівними назвами нових, раніше невідомих організацій чи фондів.
