# Flow Notes Lab 14: News Analyst Assistant

**1. Який use case обрано**
Аналіз коротких текстів українських новин (News Analyst Assistant). Класифікація та екстракція сутностей.

**2. Які етапи flow**
1. `ingest` (прийом та очищення тексту).
2. `route` (визначення, чи це новина, чи прогноз погоди/спам).
3. `execute` (виклик локальної LLM Llama-3 для екстракції категорії та сутностей).
4. `validate` (перевірка відповідності JSON-схемі).
5. `apply_fallback` (відновлення дефолтної структури даних у разі помилок LLM).
6. `export` (формування фінального JSON).

**3. Яка структура state**
Об'єкт `FlowState` містить: `case_id`, `raw_text`, `clean_text`, `route`, `routing_reason`, `extracted_data`, `validation_result`, `fallback_triggered`, `fallback_result`, `status`, масиви `errors` та `warnings`, і фінальний `final_output`.

**4. Які routes можливі**
* `news_analysis` (для цільових новин, потребує виклику LLM).
* `weather_skip` (для прогнозів погоди, пропускає виклик LLM).
* `spam_skip` (для клікбейту або спаму, пропускає виклик LLM).
* `unknown` (для порожніх або невалідних текстів).

**5. Що робить execute**
Формує prompt із суворим запитом на JSON-формат, викликає локальну модель (`unsloth/llama-3`), парсить її відповідь і записує результат у `state.extracted_data`.

**6. Що перевіряє validate**
Перевіряє, чи відповідає `extracted_data` жорсткій структурі `NEWS_ANALYSIS_SCHEMA` (за допомогою бібліотеки `jsonschema`), зокрема наявність масивів `locations`, `persons`, `organizations` та валідної категорії.

**7. Коли спрацьовує fallback**
Якщо етап `execute` завершився падінням (LLM згенерувала невалідний JSON або просто текст) або етап `validate` повернув помилку схеми. Fallback призначає категорію "General" та порожні масиви сутностей, щоб не зламати зовнішню базу даних.

**8. Який export format**
Стабільний JSON, що гарантовано містить поля: `case_id`, `route`, `final_output` (словник з даними або null), `needs_manual_review` (boolean), `status`, `warnings`, `errors`.

**9. Що flow покращив порівняно з ad-hoc pipeline (Lab 12)**
Flow повністю вирішив проблему **Unnecessary Tool Calls**. У Лаб 12 агенти марно шукали політиків у прогнозах погоди. Завдяки етапу `route` (Early Exit), Flow миттєво відкидає нерелевантні тексти. Архітектура стала стійкою до фатальних помилок завдяки валідації та ізоляції стану.

**10. Де flow був надлишковим**
Для дуже коротких і очевидних новин (з 4-5 слів) багатоступенева перевірка (Validate + Fallback) виглядає як оверінжиніринг, оскільки LLM здатна обробити їх безпомилково і без складної оркестрації.

**11. Що б ви фіксили далі**
Впровадив би динамічний Router замість хардкод-списків (`WEATHER_KEYWORDS`). Замість простого словника краще використовувати легковагову ML-модель для етапу `route`, щоб класифікація спаму чи погоди залежала від контексту, а не від точного збігу слів.
